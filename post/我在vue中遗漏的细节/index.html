组合式API以及选项式API

组合式API与选项式API的主要区别，在我看来是在标签上着重区分出来

选项式API ： 选项式api主要在script中返回data与methods，代码比较重复固定

<script>
export default {
  // data() 返回的属性将会成为响应式的状态
  // 并且暴露在 `this` 上
  data() {
    return {
      count: 0
    }
  },

  // methods 是一些用来更改状态与触发更新的函数
  // 它们可以在模板中作为事件处理器绑定
  methods: {
    increment() {
      this.count++
    }
  },

  // 生命周期钩子会在组件生命周期的各个不同阶段被调用
  // 例如这个函数就会在组件挂载完成后被调用
  mounted() {
    console.log(`The initial count is ${this.count}.`)
  }
}
</script>

组合式API：组合式api在script中添加配置setup()，使coder不用写重复多余的代码

<script setup>
import { ref, onMounted } from 'vue'

// 响应式状态
const count = ref(0)

// 用来修改状态、触发更新的函数
function increment() {
  count.value++
}

// 生命周期钩子
onMounted(() => {
  console.log(`The initial count is ${count.value}.`)
})
</script>

上述的代码相当于

<script>
import { ref, onMounted } from 'vue'

export default {
  setup() {
    const count = ref(0)

    // 用来修改状态、触发更新的函数
    function increment() {
      count.value++
    }

    // 生命周期钩子
    onMounted(() => {
      console.log(`The initial count is ${count.value}.`)
    })
        return { count,increment,onMounted }
  }
</script>

vue应用的入口

在vue应用中，我们大多数情况都会有一个App.vue文件，他会作为根组件传入createApp方法中，一般这个操作实在main.ts中完成

import { createApp } from 'vue'
import ArcoVue from '@arco-design/web-vue'
import '@arco-design/web-vue/dist/arco.css'
import ArcoVueIcon from '@arco-design/web-vue/es/icon'
import '@/assets/style/style.css'
import { createPinia } from 'pinia'

import App from './App.vue'
import router from './router'

const app = createApp(App)  ///App.vue作为根组件传入

app.use(ArcoVue)
app.use(createPinia())
app.use(ArcoVueIcon)
app.use(router)

app.mount('#app')  ///此处为css选择器

其中 app.amout('#app') 作为一个方法，将app挂载到index.html中的 id='app' 的元素上，相当于将 id='app' 的元素替换成amout方法挂载的对象

此外，我们可以通过编辑index.html来添加多个id例如 id=app1 , id=app2 ,id=...... 再通过amout方法挂载到不同的元素上，这种方法实现的每一个app都是单独的示例，每一个app都拥有自己的配置域与作用域，以此来达成多应用实例，这种方法可以在大型页面中取得较好的成效

JavaScript表达式在vue数据绑定中的作用

实际上，我们可以在任何vue数据绑定的时候使用JavaScript表达式

{{ index + 1 }}
{{ isTrue ? 'YES' : 'NO' }}
<div :id=" index-${index} " ></div>

这种使用方法仅支持单一表达式，也就是说，你不能在数据绑定的时候声明变量或者使用其他判断表达式

此外，你还可以在绑定式中调用一个组件暴露出来的方法

<time :title="toTitleDate(date)" :datetime="date">
  {{ formatDate(date) }}  
</time>

ref与reactive

ref()会接受一个任意类型的参数，并检测当参数改变时，检测与他有关联的变量，并修改这些变量在DOM中的表现

在调用ref对象绑定的数据时，应该使用 .value 属性访问对应的数据

此外，ref多用于基本数据，对应对象等复杂数据，我们使用reactive

reactive()会将变量本身变成一个响应式数据，他不需要.value 属性来调用数据，而是直接对变量本身进行调用，reactive多用于复杂数据的响应式处理

const author = reactive({
  name: 'John Doe',
  books: [
    'Vue 2 - Advanced Guide',
    'Vue 3 - Basic Guide',
    'Vue 4 - The Mystery'
  ]
})

<p>Has published books:</p>
<span>{{ author.books.length > 0 ? 'Yes' : 'No' }}</span>
